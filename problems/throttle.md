# Задача "Реализовать функцию throttle"

## Постановка 📖

Необходимо написать функцию `throttle(fn, delay)`, которая возвращает новую функцию. Эта новая функция должна ограничивать частоту вызова `fn`: исходная функция может выполняться не чаще одного раза за `delay` миллисекунд.

Если обертку вызывают много раз подряд, лишние вызовы должны игнорироваться до окончания текущего интервала.

---

## Требования ⚠️

- Первый вызов должен выполнить `fn` сразу.
- Следующие вызовы внутри интервала `delay` должны игнорироваться.
- Нужно сохранить корректный контекст `this`.
- Нужно передавать в `fn` актуальные аргументы вызова.
- Функция должна работать с любым количеством аргументов.

---

## Пример

```js
function onScroll(position) {
  console.log("Scroll:", position);
}

const throttledScroll = throttle(onScroll, 1000);

throttledScroll(10); // выполнится сразу
throttledScroll(20); // будет проигнорирован
throttledScroll(30); // будет проигнорирован

// Через 1000 мс функцию снова можно вызвать
```

---

## Решение

```js
function throttle(fn, delay) {
  let isThrottled = false;

  return function throttled(...args) {
    if (isThrottled) {
      return;
    }

    fn.apply(this, args);
    isThrottled = true;

    setTimeout(() => {
      isThrottled = false;
    }, delay);
  };
}
```

---

## Разбор решения

Главная идея — хранить флаг `isThrottled` в замыкании.

При каждом вызове `throttled`:

1. Если интервал еще не закончился, вызов игнорируется.
2. Если функция доступна для вызова, выполняется `fn`.
3. Флаг `isThrottled` переключается в `true`.
4. Через `delay` миллисекунд флаг снова становится `false`.

Метод `fn.apply(this, args)` нужен, чтобы сохранить контекст вызова и передать исходной функции все аргументы.

---

## Где используется throttle?

- Обработка события `scroll`.
- Обработка события `resize`.
- Ограничение частоты кликов по кнопке.
- Отправка аналитики не чаще заданного интервала.
- Обновление позиции курсора или drag-and-drop состояния.

---

## Оценка сложности

#### Время выполнения:

Один вызов обертки выполняется за **O(1)**, потому что используется только проверка флага и запуск таймера.

#### Пространственная сложность:

Используется **O(1)** дополнительной памяти для хранения флага `isThrottled`.

---

## Тесты

```js
const log = throttle((value) => {
  console.log(value);
}, 1000);

log("first"); // выполнится
log("second"); // проигнорируется
log("third"); // проигнорируется

setTimeout(() => {
  log("fourth"); // выполнится
}, 1100);
```
