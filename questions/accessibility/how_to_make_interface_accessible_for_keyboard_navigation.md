# Как сделать интерфейс доступным для клавиатурной навигации?

---

Доступная клавиатурная навигация означает, что пользователь может пройти основной сценарий без мыши: перейти к элементам, понять где находится фокус и выполнить нужные действия.

---

#### Использовать нативные элементы

Нативные интерактивные элементы уже поддерживают клавиатуру.

```html
<button type="button">Сохранить</button>
<a href="/profile">Профиль</a>
<input aria-label="Поиск" />
```

Лучше использовать `<button>` вместо кликабельного `<div>`, потому что кнопка уже работает с `Tab`, `Enter` и `Space`.

---

#### Видимый фокус

Пользователь должен видеть, какой элемент сейчас активен.

```css
.button:focus-visible {
  outline: 2px solid #2563eb;
  outline-offset: 2px;
}
```

Нельзя просто убирать `outline`, не добавив понятную замену.

---

#### Логичный порядок перехода

Порядок фокуса должен соответствовать визуальной структуре страницы. Обычно достаточно правильного порядка элементов в HTML.

`tabindex` стоит использовать осторожно. Значения больше `0` часто усложняют навигацию и делают интерфейс непредсказуемым.

---

#### Итог

Для клавиатурной доступности важны нативные элементы, видимый фокус, логичный порядок перехода и отсутствие ловушек фокуса. На собеседовании хорошо упомянуть, что интерфейс нужно проверять вручную клавишами `Tab`, `Shift + Tab`, `Enter`, `Space` и `Esc`.
