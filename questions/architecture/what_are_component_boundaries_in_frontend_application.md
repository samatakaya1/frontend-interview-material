# Что такое компонентные границы во frontend-приложении?

---

Компонентные границы показывают, за что отвечает компонент, какие данные он принимает и какие детали скрывает внутри себя.

Хорошие границы помогают не смешивать отображение, состояние, бизнес-логику и работу с внешними сервисами в одном месте.

---

#### Пример

```jsx
function UserCard({ user, onEdit }) {
  return (
    <article>
      <h3>{user.name}</h3>
      <button onClick={() => onEdit(user.id)}>Edit</button>
    </article>
  );
}
```

Компонент `UserCard` отвечает за отображение карточки пользователя. Он не знает, откуда пришел `user`, как открывается форма редактирования и как сохраняются данные.

---

#### Что важно знать?

- Компонент должен иметь понятную ответственность.
- Внешний API компонента — это props, events/callbacks и ожидаемое поведение.
- Детали реализации лучше прятать внутри компонента или рядом с ним.
- Не стоит передавать в компонент слишком много несвязанных props.
- Если компонент одновременно делает запросы, управляет сложным состоянием и рисует много UI, границы, вероятно, размыты.
- Хорошие границы упрощают тестирование, переиспользование и замену реализации.

---

#### Итог

Компонентные границы помогают строить frontend-приложение из понятных частей. На интервью важно объяснить, что компонент должен скрывать внутренние детали и давать наружу простой, предсказуемый интерфейс.
