# Чем отличаются `git reset` и `git revert`?

---

`git reset` и `git revert` помогают отменять изменения, но делают это по-разному.

Главное отличие: `reset` двигает указатель ветки и может переписывать историю, а `revert` создает новый коммит, который отменяет изменения старого коммита.

---

#### `git reset`

`git reset` переносит текущую ветку на другой коммит.

```bash
git reset --hard HEAD~1
```

После такого reset последний коммит исчезает из истории текущей ветки. Это удобно для локальных правок, которые еще не были опубликованы.

Есть разные режимы:

- `--soft` оставляет изменения в staged.
- `--mixed` оставляет изменения в рабочей директории.
- `--hard` удаляет изменения из рабочей директории.

---

#### `git revert`

`git revert` не удаляет старый коммит, а создает новый коммит с обратными изменениями.

```bash
git revert abc123
```

История остается линейной и понятной: видно, какой коммит был отменен.

---

#### Главное правило

`reset` безопаснее использовать для локальной истории, которую еще никто не получил. `revert` лучше использовать для опубликованной истории, особенно в командной работе.

---

#### Итог

`git reset` переписывает историю, а `git revert` добавляет новый коммит отмены. На собеседовании важно показать понимание, что выбор зависит от того, опубликованы ли изменения и работает ли с веткой команда.
