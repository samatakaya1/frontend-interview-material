# Что такое замыкание (Closure) в JavaScript?

---

**Замыкание (Closure)** в JavaScript — это возможность функции запоминать свое лексическое окружение, даже если эта функция вызывается вне области видимости, где она была создана.

Проще говоря: если внутренняя функция использует переменные внешней функции, то она сохраняет доступ к этим переменным после завершения работы внешней функции.

---

#### Пример

```js
function createCounter() {
  let count = 0;

  return function increment() {
    count += 1;
    return count;
  };
}

const counter = createCounter();

console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

Функция `createCounter` завершила выполнение после строки `const counter = createCounter()`, но функция `increment` продолжает иметь доступ к переменной `count`. Это и есть замыкание.

---

#### Почему это работает?

Когда функция создается, JavaScript сохраняет ссылку на ее лексическое окружение — место, где функция была объявлена.

В примере выше функция `increment` была объявлена внутри `createCounter`, поэтому она помнит переменную `count`. Переменная не удаляется из памяти, пока на внутреннюю функцию остается ссылка.

---

#### Где используются замыкания?

- **Инкапсуляция данных:** можно скрыть переменные от внешнего кода.
- **Функции-фабрики:** можно создавать функции с заранее сохраненным состоянием.
- **Обработчики событий:** callback может помнить данные из места, где был создан.
- **Асинхронный код:** функции внутри `setTimeout`, `Promise` или `fetch` могут использовать переменные внешней области.
- **Каррирование и частичное применение:** функция может запомнить часть аргументов и вернуть новую функцию.

---

#### Частая ошибка на собеседовании

Неверно говорить, что замыкание — это просто "функция внутри функции". Вложенная функция сама по себе еще не объясняет суть. Замыкание появляется тогда, когда функция сохраняет доступ к переменным из внешней области видимости.

Пример:

```js
function outer() {
  const message = "Hello";

  return function inner() {
    console.log(message);
  };
}

const sayHello = outer();

sayHello(); // "Hello"
```

Функция `inner` вызывается уже после выполнения `outer`, но все еще использует переменную `message`.

---

#### Что важно сказать на интервью?

Хороший ответ должен включать три идеи:

- Замыкание связано с лексической областью видимости.
- Функция сохраняет доступ к переменным внешней функции.
- Эти переменные продолжают существовать, пока на функцию есть ссылка.

---

#### Итог

Замыкание — один из ключевых механизмов JavaScript. Оно позволяет функциям хранить состояние, создавать приватные данные и строить гибкие абстракции. Именно поэтому вопрос о замыканиях часто задают на собеседованиях: он показывает, понимает ли кандидат, как JavaScript работает с функциями, областями видимости и памятью.
